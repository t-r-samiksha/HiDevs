## SECTION 8: FRONTEND–BACKEND INTEGRATION GAPS

1. **`admin/prompts` is static** — `settings/intelligence` "versioned prompt editor" reads a hardcoded array and PATCH doesn't persist. Only real gap of "backend looks real but isn't."
2. **`setup-db` RPC is a stub** — `POST` can't actually run DDL; schema changes require pasting `GET /api/setup-db` SQL into Supabase. Several features degrade gracefully until migrations run: `escalation_logs.run_id` (durable HITL resume), `rooms.created_by`, `projects.description`, `items.enkrypt_checks`.
3. **`webhooks/jibri` orphaned in practice** — no Jibri deployed; recording uses the browser path instead. Route exists, never called.
4. **Two ingestion paths for recordings** — `/api/meetings/record` (deterministic, 1 Gemini call, items marked `pending_review`, no Enkrypt/Qdrant depth) vs `/api/pipeline` (full depth, ~8 Gemini calls). Recordings intentionally use the light path → recorded-meeting items lack trust scores/embeddings.
5. **`pipelineSupervisorWorkflow` is not wired into the main upload flow** — it's only reachable via `/api/pipeline/supervise`; the upload page still uses `/api/pipeline`.
6. **`GET /api/meetings` requires `project_id`** but the meetings page uses the Supabase client directly (no filter) — inconsistent but not broken (page works).
7. **Prior AUDIT.md `project_id` bugs — FIXED**: dashboard insights/search/reports now pass `project_id`/consolidated correctly (verified in earlier sessions).

## SECTION 10: CRITICAL ISSUES (by severity)

**CRITICAL (can fail the live demo)**
- **Gemini 20 req/day free-tier quota.** Any live upload/ask/insight/draft may 429. → Mitigations: fresh Google project/key before demo, pre-seed data, screen-record the LLM moments, and lean on quota-safe flows live.
- **Recording depends on browser mic/screen permission + no ad-blocker** (Jitsi `external_api.js` is blocklisted). Practice on the demo machine.

**HIGH (judges will notice)**
- **`admin/prompts` static** — the "prompt editor" doesn't actually edit. Either wire it to a `prompts` table or hide the edit affordance.
- **Recorded-meeting items lack trust scores/embeddings** (light path) — they show `pending_review` and won't appear in semantic search. Inconsistent with uploaded meetings.
- **Migrations not auto-applied** — if `run_id`/`created_by`/`description`/`enkrypt_checks` columns aren't added, those features silently degrade. Run the `GET /api/setup-db` SQL once.

**MEDIUM**
- Production pipeline isn't the registered Mastra workflow (Section 1).
- `Helm/src/mastra` orphaned — dead weight; risks confusing judges who inspect the repo.
- Single golden transcript; closed loop stops at capture.
- Two divergent recording ingestion paths.

**LOW**
- `webhooks/jibri` dead route.
- Duplicate Enkrypt helper logic across pipeline + workflows.
- `setup-db` RPC stub messaging.
